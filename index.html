Projects


Towards an Affective Video Recommendation System


My 10-week participation in the Computational Sensing REU at RIT culminated with a paper we titled "Towards an Affective Video Recommendation System." In this context, affective refers to data about people's emotions. The goal of this project was to obtain affective data from subjects while they watched video clips and later find a link between this data and how they rated the videos.


We had people watch a series of 5 different videos meant to evoke different emotions while their behaviour was beeing recorded by two webcams. We processed the stream from one webcam through the Affectiva module in iMotions, a facial analysis software. Simultaneously, we used the second webcam to estimate people's pulse through a technique known as photoplethysmography.


For illustration purposes, we combined the facial analysis data with the pulse data into what we called Affective Graphs. These graphs map the user's facial expressions and pulse as a function of time. For more information, download my poster presentation or the full paper.


Job Search App


As part of a final project for my Mobile Design class, I designed and implemented a hybrid mobile app to search for computing jobs. The app allows users to look for jobs through the Github Jobs API. Users can search by keyword or by location. The app can also use your current location to look for jobs around you.


I used the Ionic framework to build this app. Ionic is a framework built on top of Cordova that allows for a fast development of high-quality mobile apps using web technologies such as Angular, Typescript, HTML, and SCSS. Cordova provides plugins that enable OS-specific functionality like Geolocation, which I used in my app. Ionic provides the front-end and follows the Model-View-Controller pattern, separating concerns and enforcing modular code.


BrickDash


BrickDash is a web implementation of the board game BalderDash. This game is the product of 24 hours of work at RIT's BrickHack4 event, hence the name of the game. Clever, right? My friends and I chose to work on this web application to avoid the constant paper waste generated by the original game.


We made the front-end using HTML, CSS, and Javascript. My main responsibility was the backend, which I wrote in PHP. We hosted the site on a WAMP server I set up on my computer, which allowed us to test it as we made progress.


The web application allows for any number of players to join and play a game from their computers or smartphones, completely removing the need for paper. Furthermore, it doesn't require user registration. Instead, users can join each other by sharing a token provided by the game, making for an easy-to-start experience.
